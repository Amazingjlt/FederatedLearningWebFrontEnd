联邦学习平台用例图结构化分析（用于绘制 Use Case Diagram）

一、参与者（Actors）
1. 平台管理员（Admin）
职责：负责平台运维与治理，拥有系统全量管理权限。
可参与能力：仪表盘查看、作业全生命周期管理、模型管理、客户端管理、数据质量分析、系统设置与用户账户管理。

2. 研究员/普通业务用户（User/Researcher）
职责：负责联邦学习任务发起、训练监控、模型查看与数据分析。
可参与能力：仪表盘查看、作业管理、模型管理、客户端查看、数据质量分析。
受限能力：通常不具备系统级设置与用户管理权限。

3. 平台后端服务（NVFLARE Backend / API Server，外部系统）
职责：提供业务接口、鉴权校验、任务执行状态与统计数据。
交互方式：前端通过 HTTP API 调用（如 /job/*、/model/*、/settings/* 等）。

4. 联邦客户端节点（FL Client Node，外部实体）
职责：参与联邦训练、上报状态与质量相关信息。
交互方式：被平台纳管（接入/删除/重连），其状态和质量数据在前端可视化展示。

二、系统边界（System Boundary）
系统名称：联邦学习平台前端系统（FederatedLearningWebFrontEnd）
边界内功能域（建议在用例图中分包/分区）：
1. 仪表盘模块（Dashboard）
2. 作业管理模块（Job Management）
3. 模型管理模块（Model Management）
4. 客户端管理模块（Client Management）
5. 数据质量分析模块（Data Quality）
6. 系统设置模块（Settings）
7. 认证与权限协同（Token携带、未授权处理、角色访问约束）

四、关系（Relationships）
1. 参与者与用例关联（Association）
平台管理员 -> 关联所有模块用例。
研究员/普通业务用户 -> 关联仪表盘、作业管理、模型管理、客户端查看、数据质量分析。
平台后端服务 -> 与全部“数据获取/提交/保存/导出”类用例关联（作为外部支持系统）。
联邦客户端节点 -> 与“客户端管理”“作业执行状态”“数据质量节点展示”相关用例间接关联。

2. 用例包含关系（<<include>>，建议画图时使用）
查看作业详情 <<include>> 查看作业指标曲线。
查看作业详情 <<include>> 查看作业日志。
保存系统设置 <<include>> 查看系统设置（读取当前配置）。
新增用户 <<include>> 表单校验。
编辑用户 <<include>> 表单校验。
生成数据质量报告 <<include>> 按时间与阈值筛选质量数据。
查看客户端详情 <<include>> 查看客户端运行指标图。

3. 用例扩展关系（<<extend>>，建议画图时使用）
中止运行中作业 <<extend>> 查看作业详情（仅作业状态为“运行中”时出现）。
加载更多警告 <<extend>> 查看警告列表（当未加载完全部告警时触发）。
查看节点质量详情 <<extend>> 查看节点3D质量分布（点击节点触发）。
测试后端连接 <<extend>> 配置后端连接参数（完成参数填写后触发）。
上传连接证书 <<extend>> 配置后端连接参数（仅启用证书场景时触发）。
重新连接客户端 <<extend>> 查看客户端详情（节点离线/异常时触发）。
模型比较 <<extend>> 查询模型列表（勾选>=2个模型时触发）。

4. 参与者泛化关系（Generalization，按论文需要可选）
平台用户（抽象参与者）
-> 平台管理员
-> 研究员/普通业务用户
说明：管理员继承平台用户的通用能力，并扩展系统设置与用户管理权限。

五、建议的用例图绘制方式（可直接落图）
1. 顶层图：1个系统边界 + 4个参与者 + 6大模块核心用例。
2. 分模块子图：为每个模块单独绘制子用例图，补充 include/extend 细节。
3. 权限图：单独补一张“角色-用例矩阵图”或在用例图上用注释标明 admin/user 的可用范围。

六、可直接使用的角色-用例简表
1. Admin：全部用例。
2. User/Researcher：除“系统设置、用户增删改”外的大部分业务用例。
3. Backend：支撑所有接口型用例（查询、保存、提交、导出、测试连接）。
4. FL Client Node：支撑客户端状态、作业执行反馈、质量节点数据相关用例。
